## 📌 Project Overview
This project performs an end-to-end data engineering and business intelligence analysis on the **Olist E-commerce dataset** (Brazilian marketplace). 
Using a **Medallion Architecture** on **Azure Synapse Analytics**, I transformed raw data into actionable business insights focused on logistics optimization and seller performance.

## 🛠️ Tech Stack
* **Storage:** Azure Data Lake Storage (ADLS) Gen2
* **Processing:** PySpark (Azure Synapse Spark Pool)
* **Data Modeling:** Medallion Architecture (Bronze -> Silver -> Gold)
* **Analytical Layer:** Serverless SQL (Azure Synapse)
* **Visualization:** Power BI / Matplotlib

## 📂 Repository Structure
* `notebooks/`: PySpark scripts for data ingestion, cleansing, and type casting.
* `sql/`: Logical data warehouse setup and deep-dive business queries.
* `images/`: Data visualizations and architecture diagrams.

### 📈 Business Deep-Dive
<details>
1. Seller Performance
Analysis: Used Pareto (80/20) analysis to evaluate marketplace resilience.

Finding: Discovered that Olist has a very healthy "Long-tail" market where the top 20% of GMV is spread across over 500+ sellers, meaning low platform dependency on specific "Super-Sellers".

Strategic Suggestion: Instead of aggressive individual account management, the platform should focus on automated policy enforcement and scaling seller tools, as the revenue is decentralized.

2. Logistics Efficiency 
Analysis: Data reveals that remote regions have lower delay rates than core metropolitan areas.

Finding: This is driven by overly conservative "Estimated Delivery Date" (EDD) algorithms for remote zones.

Strategic Recommendation: Olist should recalibrate the EDD for core regions to reduce "false-positive" delays and improve customer trust in urban markets.

3. Payment & Satisfaction 
Analysis: Correlated payment_installments with review_score.

Finding: There is a slight negative correlation between the number of installments and satisfaction scores, regardless of delivery speed.

Strategic Suggestion: Launch a "Post-Purchase Engagement" campaign for customers on 6+ month payment plans. By sending maintenance tips or exclusive discount vouchers during the 3rd or 4th month of repayment, the platform can mitigate the "payment pain" and improve long-term NPS.
</details>

## 🚀 How to Run
1. Configure an Azure Storage Account and upload the Olist CSV files.
2. Run the Pyspark notebooks in `notebooks/` to populate the Silver layer.
3. Use the SQL scripts in `sql/` to create views and run analytical queries.
